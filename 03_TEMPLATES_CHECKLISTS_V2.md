# CHECKLIST PACK — Execution Checklists (Version 2)

**Version:** V2 (Full)  
**Last updated:** 2026-02-24

---

## 1) Onboarding — new landlord + property
- [ ] Management authority saved to Property Tree docs (01 Management & Owner Docs)
- [ ] Owner record created (bank details verified)
- [ ] Property record created (address verified)
- [ ] Fees/disbursement/statement settings applied
- [ ] Property Tree docs folders created
- [ ] Keys/access plan documented
- [ ] TPS property setup completed (compliance fields entered)
- [ ] Owner instructions recorded (approval style, quote-first, spend threshold)
- [ ] Compliance evidence gathered OR gaps converted into Tapi action plan
- [ ] Owner welcome email sent

---

## 2) Leasing — Trade Me → TPS BookMe → TPS applications
- [ ] Trade Me listing published
- [ ] Trade Me listing ID entered into TPS property
- [ ] BookMe viewings created and visible
- [ ] Tenant-in-place access confirmed (if applicable)
- [ ] Applications received and triaged
- [ ] Checks run on preferred applicant(s) only
- [ ] Owner recommendation sent and decision captured in writing
- [ ] Application approved in TPS
- [ ] Agreement generated and sent for signing in TPS

---

## 3) Tenancy setup — TPS → bond → Property Tree
- [ ] Agreement fully signed and exported
- [ ] Bond + initial rent collected to trust
- [ ] Bond lodged via TPS and paid to Tenancy Services with reference
- [ ] Bond confirmation saved to Property Tree
- [ ] Tenancy created in Property Tree with perfect dates/rent
- [ ] Agreement stored in Property Tree docs
- [ ] Welcome pack sent
- [ ] Ingoing inspection scheduled

---

## 4) Routine inspections (quarterly)
- [ ] Inspection scheduled in Property Tree and sync confirmed to Inspection Express
- [ ] Email sent ~10 days out
- [ ] SMS sent ~9 days out asking for confirmation
- [ ] Tenant confirmed in writing
- [ ] Property Tree updated to Confirmed
- [ ] 24-hour reminder sent
- [ ] Inspection completed and report published
- [ ] Report sent to landlord and tenant
- [ ] Maintenance items created and synced to Tapi (if needed)
- [ ] Follow-ups tracked to completion

---

## 5) Maintenance (Tapi-first)
- [ ] Request received in Tapi and tenant acknowledged
- [ ] Urgency assessed (urgent vs routine)
- [ ] Owner approval captured for routine work
- [ ] Quote-first used where needed (> $500 or uncertain)
- [ ] Work order/quote request issued with scope/access/WO#
- [ ] Job tracked and updates sent to tenant/owner
- [ ] Invoice matched and approved; coded correctly (owner/tenant/DIY)
- [ ] Invoice synced to Property Tree for payment run
- [ ] Job closed and key artifacts filed

---

## 6) Renewals (fixed term)
- [ ] Landlord contacted at ~90 days
- [ ] Rent evidence prepared (Valua) if required
- [ ] Landlord decision captured in writing
- [ ] Tenant notified promptly once owner direction confirmed
- [ ] Follow-ups scheduled (42 days, 14 days, escalation call if needed)
- [ ] TPS renewal/extension executed if applicable
- [ ] Property Tree updated and docs filed

---

## 7) End of tenancy / lease break
- [ ] End date confirmed in writing
- [ ] Vacate pack sent and filed
- [ ] Outgoing inspection completed and filed
- [ ] Bond process actioned and outcome filed
- [ ] Lease break workflow (if applicable) followed and documented
- [ ] Re-listing started if needed
