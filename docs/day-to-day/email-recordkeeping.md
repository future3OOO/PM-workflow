# Email & Recordkeeping Rules

**Version:** V2.0  
**Last updated:** 2026-02-24

---

## Goal
Make email an operational tool, not a black hole: the VA closes loops, decisions are captured, and the Property Tree property record stays complete.

## Core rules
- Every actionable email has:
  - a property label, and
  - a status label.
- Anything critical is filed into Property Tree docs (PDF if needed).
- If a decision affects money, tenancy dates, compliance, inspections, or maintenance: it must be traceable (email/Tapi/TPS) and filed.

## When to use email vs in-system messages
- Use TPS for leasing and agreement communications where it naturally supports the workflow.
- Use Tapi for maintenance updates/approvals/work orders/invoice messaging.
- Use email for:
  - exceptions, escalations, complex negotiation
  - where you need a clear written confirmation outside the platform
  - when a party is not responding via platform

## “Close the loop” definition
A thread is only closed when:
- the action is completed (or scheduled), AND
- the record is updated (Property Tree/TPS/Tapi), AND
- the evidence is filed (Property Tree docs).

!!! tip "Golden Rule"

    If a decision affects **money, tenancy dates, compliance, inspections, or maintenance** — it must be traceable (email / Tapi / TPS) **and** filed to Property Tree docs.
