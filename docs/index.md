## How This Library Works

| Type | What it covers | When to use it |
|---|---|---|
| **[Playbooks](playbooks/landlord-property-onboarding.md)** | Lifecycle process — *what* happens and *when* | Starting or following a workflow end-to-end |
| **[SOPs](sops/propertytree-create-owner-property.md)** | Click-by-click system instructions — *how* to do it | Executing a specific task in a specific system |
| **[Templates](templates/notices-emails.md)** | Copy/paste emails, notices, and website copy | Sending a communication or filling in a form |
| **[QA Checklists](qa/daily-triage.md)** | Daily, weekly, and monthly quality controls | Making sure nothing falls through the cracks |

!!! tip "Golden Rule"
    If it isn't recorded in **Property Tree** (or linked into the property record), it didn't happen.

## Systems Covered

| System | Role |
|---|---|
| **Property Tree** | System of record — master data, tenancy dates, trust accounting, document repository |
| **TPS / Tenancy.co.nz** | Leasing, agreements, renewals, compliance, applications, bond workflow |
| **Trade Me** | Marketing channel — listings and enquiry routing |
| **Tapi** | Maintenance CRM — intake, triage, work orders, invoices, Property Tree sync |
| **Inspection Express** | Inspection execution, reporting, maintenance item sync to Tapi |
| **Valua** | Rent evidence and recommendations |
| **Shortwave AI** | AI email client for drafting owner/tenant communications |
