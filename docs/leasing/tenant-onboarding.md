# Tenant Onboarding

*V2 — Last updated 2026-02-24*

---

## Goal
Give tenants a smooth start with clear instructions and a complete ingoing condition record, with everything filed to Property Tree.

## Trigger
Agreement signed; start date set.

## Outputs (definition of done)
- Welcome pack sent pre-start
- Ingoing inspection completed, published, and stored in Property Tree
- Tenant understands maintenance pathway (website form → Tapi) and urgent escalation
- Key handover recorded

---

## Stage 1 — Welcome pack (send before start date)

!!! tip "Timing"
    Send the welcome pack before the tenancy start date so the tenant has time to read it.

Include:
- key handover process
- rent payment instructions and first due date
- maintenance request instructions (your website form) and urgent guidance
- inspection expectations (quarterly) and tenant responsibility basics
- ingoing condition report timeline for tenant feedback

Template: [Notice & Email Templates](../day-to-day/notice-email-templates.md)

---

## Stage 2 — Ingoing inspection (Property Tree → Inspection Express)
- schedule ingoing inspection in Property Tree; confirm sync in Inspection Express
- complete inspection thoroughly with photos
- publish report; send to tenant and owner; store in Property Tree docs

SOP: [Inspection Express](../inspections/inspection-express.md)

---

## Stage 3 — First week check
- confirm payments and contact details are correct
- capture any immediate issues into Tapi if required

---

## Recordkeeping
- welcome pack copy (email PDF if needed) filed to Property Tree
- ingoing report filed to Property Tree
