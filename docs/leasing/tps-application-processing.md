# Application Processing (TPS)

**Version:** V2.5  
**Last updated:** 2026-04-01

---

## Purpose / Outcome
Applications are processed consistently with a documented decision trail and appropriate use of checks.

Part of: [Leasing Lifecycle](leasing-lifecycle.md) → Stage 5

## Rules

!!! warning "Background checks — privacy rule"
    Run background/credit checks on **preferred applicant(s) only** — never on all applicants. Capture the owner's decision in writing before proceeding.

- Triage all applications for completeness.
- Approve in TPS only after the owner decision is captured and the successful-applicant workflow is ready to be completed in sequence.

## Steps
<span class="pp-verified-label">Verified from video analysis</span>

### 1) Triage and shortlist

Navigate to: **TPS → Applications / My Applicants**

- Review applications for completeness
- Shortlist the preferred applicant(s)
- Run checks only on the preferred applicant(s)

### 2) Get and file the owner's decision

- Provide the owner recommendation summary
- Capture the owner's acceptance in writing before progressing

### 3) Mark the successful application in TPS

Navigate to: **TPS → My Applicants → select successful applicant → Update applicants**

- Set the applicant to **Application Successful**

### 4) Create the draft agreement before sending the acceptance letter
<span class="pp-verified-label">Verified from video analysis</span>

- Open the draft tenancy agreement workflow before sending acceptance
- TPS generates the payment **reference** inside the agreement / move-in-cost workflow
- Copy that reference into the acceptance communication
- This reference is also used later for payment matching in Property Tree

### 5) Review and send the acceptance communication

- Confirm the move-in date with the tenant rather than blindly using the date originally entered on the application
- Check the standard term being offered, usually **12 months fixed**
- Check the move-in figures carefully:
  - first week rent
  - general bond
  - pet bond if applicable
- Keep the PM copied on the acceptance email
- If the property is keyless or uses a different access process, replace the default key-collection wording with the correct instructions

### 6) Clean up the remaining applicants

- Archive stale or duplicate applications where appropriate
- Bulk decline the unsuccessful applicants
- Make sure the successful applicant is not included in the decline action

### 7) Keep marketing live until the agreement is properly completed

- Do not withdraw the Trade Me listing just because the application is approved
- Leave the listing live until the tenant has signed and the PM has countersigned the agreement

### 8) File the decision trail

- File the decision summary and any approval evidence to Property Tree as permitted by your recordkeeping standard
